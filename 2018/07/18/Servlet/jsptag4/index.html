    简单标签简介#

    传统标签的开发中，我们发现处理过程还是比较复杂的，开发人员必须要考虑到doStartTag方法和doEndTag方法的返回值，为了简化开发标签的过程，JSP2.0引入了一种新的标签扩展机制。

    SimpleTag接口详解#

    方法详解#

    在javax.servlet.jsp.tagext.SimpleTag接口中定义了5个方法

    void setJspContext(JspContext pc):由servlet容器调用该方法，向SimpleTag对象传递当前的JspContext对象，其中JspContext类是PageContext类的父类，

    void setParent(JspTag parent):由Servlet容器调用该方法，向当前的SimpleTag对象传递负标签的JspTag对象

    JspTag getParent():返回父标签的JspTag对象

    void setJspBody(JspFragment jspBody):由Servlet容器调用该方法，向当前SimpleTag对象中传递标签主体，参数jspBody表示当前标签的主体，它封装类一段jsp代码

    void doTag():该方法负责具体的标签处理过程，与传统标签处理类相比，doTag方法并没有返回值

    执行流程#

    simpleTag对象由Servlet容器负责创建，每次当Servlet容器执行jsp文件时，遇到jsp文件中的自定义简单标签时，都会创建一个SimpleTag对象，当标签处理完毕后，就会销毁该SimpleTag对象，这一点与传统标签不一样，传统标签则会由容器来缓存标签处理类的实例，以便重复使用该标签

    Servlet容器在得到SimpleTag对象后，会安装以下流程调用相关的方法

    1.Servlet容器调用Simple对象的setJspContent方法和setParent方法，把当前的jsp页面的JspContext对象以及父标签处理对象传递给当前的SimpleTag对象，如果不存在父标签，则把父标签处理对象设置为null

    2.Servlet容器调用SimpleTag对象的一系列set方法，设置SimpleTag对象的属性，如果标签没有属性，则无需这个步骤

    3.如果标签存在标签主体，Servlet容器就调用SimpleTag对象的setJspBody方法，设置标签主体

    4.servlet容器调用SimpleTag对象的doTag方法，在该方法中完成处理标签的具体逻辑
